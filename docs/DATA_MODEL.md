# ALMAS Framework
# Data Model v1.1

---

# Philosophy

ALMAS does not store files.

ALMAS stores knowledge.

Every source (YouTube, PDF, Website, Voice, Image...) becomes one unified Knowledge object.

---

# Universal Knowledge Object

```json
{
  "id": "",
  "type": "",
  "title": "",

  "summary": "",

  "keyPoints": [],

  "tags": [],

  "ideas": [],

  "tasks": [],

  "source": {
    "url": "",
    "author": "",
    "duration": "",
    "createdAt": ""
  },

  "status": "approved",

  "createdAt": "",
  "updatedAt": ""
}
```

---

# Supported Types

- youtube
- pdf
- website
- voice
- image
- idea
- note
- book
- article

---

# Status

- pending
- approved
- archived
- deleted

---

# Required Fields

Every Knowledge object must contain:

- id
- type
- title
- summary
- source
- status
- createdAt
- updatedAt

---

# AI Generated Fields

These fields are automatically generated by AI.

- keyPoints
- tags
- ideas
- tasks

---

# Search Strategy

Search is performed across:

- title
- summary
- keyPoints
- tags

Future versions will also support:

- ideas
- tasks
- semantic embeddings
